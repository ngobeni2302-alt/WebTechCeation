=============================================================================
                          TechMINDS - Project Explanation
=============================================================================

This document provides a comprehensive overview of the TechMINDS project, 
explaining the technologies used, how they work together, and a file-by-file 
breakdown of the codebase.

=============================================================================
1. TECHNOLOGIES USED
=============================================================================

FRONTEND TECHNOLOGIES:
- HTML5: Used to create the structural foundation and semantic meaning of the web pages.
- Vanilla CSS3 / SCSS: Used for styling the website. Variables (CSS Custom Properties) are heavily utilized to implement the dynamic Dark/Light theme switching. 
- Vanilla JavaScript (ES Modules): Used for interactive logic without relying on heavy frameworks like React or Angular. ES Modules (`import`/`export`) are used to break the code into maintainable, distinct files.
- Font Awesome: Used for scalable vector icons throughout the interface.
- Google Fonts (Outfit): Provides modern typography to enhance the premium feel of the design.

BACKEND TECHNOLOGIES:
- Python 3: The core programming language for the backend server.
- FastAPI: A modern, fast (high-performance) web framework for building APIs with Python 3 based on standard Python type hints.
- SQLAlchemy: An Object Relational Mapper (ORM) for Python. It allows the Python code to interact with the database using Python objects instead of raw SQL queries.
- SQLite: A lightweight, serverless database engine used to store data locally in a single file (`users.db`).
- PyJWT: A library used to generate and verify JSON Web Tokens (JWTs) for secure user authentication.
- SlowAPI: Used for rate-limiting API endpoints to prevent brute-force attacks.
- Pydantic: Data validation and settings management using Python type annotations.
- python-dotenv: Used to load environment variables from the `.env` file securely.

=============================================================================
2. HOW THE TECHNOLOGIES WORK TOGETHER
=============================================================================

The architecture of this project is a "Single Page Application (SPA)" style frontend connected to a "RESTful API" backend.

1. The Web Server: 
When you run the server, the application uses FastAPI to listen for incoming network requests on port 8080. 

2. Serving the Frontend:
If you request the homepage (`http://localhost:8080`), the Python server reads `index.html` and sends it to the browser. The browser then requests the linked CSS and JavaScript files. 

3. Client-Side Interactivity:
The frontend JavaScript is broken into smaller modules (`auth.js`, `theme.js`, etc.). These scripts attach event listeners to buttons and forms. For example, when you click the "Theme Toggle", JavaScript swaps CSS variables to change colors instantly.

4. API Communication & Authentication:
When a user attempts to log in via the Admin Modal, `auth.js` intercepts the form submission and uses the JavaScript `fetch()` API to send a POST request with the credentials to `/api/login`.
The Python server receives this request, parses the JSON payload using Pydantic, and queries the SQLite database using SQLAlchemy. If the credentials match, Python uses PyJWT to create an encrypted token and sends a success response. The backend also sets an HTTP-only, secure cookie containing the access token to prevent XSS attacks. The frontend updates the UI based on the success response. 

5. Error Handling:
If the API request fails (e.g., incorrect password, server crash), the server responds with specific HTTP status codes (like 401 Unauthorized or 500 Server Error). The frontend's `errorHandler.js` intercepts these codes and triggers the appropriate UI response (e.g., highlighting input boxes in red or showing a "Session Expired" alert).

=============================================================================
3. FILE-BY-FILE BREAKDOWN
=============================================================================

--- ROOT DIRECTORY ---
- `index.html`
  The main entry point for the frontend. It contains all the semantic HTML, the layout, navigation, the Admin login modal, and includes meta tags for SEO. 

- `README.md`
  Standard markdown file containing instructions on how to install dependencies and run the project.

- `.env`
  A secure file (ignored by version control) that stores environment variables like database URLs or secret keys used for JWT encryption.

- `server_logs_explanation.txt`
  A plain text file explaining the normal, diagnostic output you see in the terminal when launching the server.

--- BACKEND DIRECTORY (`/backend/`) ---
- `server.py`
  The core server file. It creates a FastAPI application that serves static files (HTML, CSS, JS) and defines specific routes (like `/api/login` and `/api/signup`) with built-in rate limiting and data validation to handle backend business logic.

- `database.py`
  Handles the configuration and connection to the SQLite database. It initializes the SQLAlchemy engine and creates a session maker so the server can talk to the database.

- `models.py`
  Defines the database schema using Python classes. For example, it defines a `User` class with columns for ID, username, password, etc., which SQLAlchemy translates into SQL tables.

- `init_db.py`
  A utility script meant to be run once to initialize the database. It tells SQLAlchemy to create the tables defined in `models.py` and populate them with default/admin users.

- `users.db`
  The actual SQLite database file. All user data is stored persistently inside this file.

--- FRONTEND JAVASCRIPT (`/assets/js/`) ---
- `main.js`
  The main entry point for the frontend JavaScript. It imports all other modules and initializes them when the HTML document is fully loaded.

- `auth.js`
  Contains the logic for the Admin Login modal. It listens for form submissions, sends authentication requests to the backend via `fetch()`, handles session storage, and manages password visibility toggling.

- `errorHandler.js`
  A generic API utility that parses HTTP responses and triggers UI actions. It interprets status codes (401, 422, 500) and displays toast notifications or highlights specific invalid input fields in red.

- `nav.js`
  Handles navigation interactivity. It manages the mobile hamburger menu toggle and smooth scrolling when clicking on navbar links.

- `theme.js`
  Contains the logic for the light/dark mode switch. It updates an HTML attribute (`data-theme`) which triggers the CSS file to swap color variables.

- `ui.js`
  Handles general visual enhancements. This includes setting up Intersection Observers for scroll animations (making elements fade in as you scroll down) and managing the image expansion modal logic.

--- FRONTEND STYLES (`/assets/css/` & `/src/scss/`) ---
- `assets/css/main.css`
  The compiled, minified CSS file that the browser reads. It contains all styling, including dark/light mode variables, grid layouts, animations, and the newly added `.input-error` styles.

- `src/scss/` (Directory)
  Contains the source SCSS files. SCSS is a preprocessor that allows developers to write CSS more efficiently using nesting and variables. This code is compiled into `main.css`.

--- SCRIPTS DIRECTORY (`/scripts/`) ---
- `open_web.sh`
  A convenient Bash shell script designed for Linux/Mac users. It checks for dependencies, launches `server.py` in the background, and opens the default web browser to the correct localhost URL.
